# Hasura DDN Documentation

Marketingish CTA...

<CodeStep
language="bash"
code={`npm i ? -g`}
>

## Install the Hasura CLI

Install the new Hasura CLI globally using npm.

</CodeStep>

<CodeStep
language="bash"
code={`hasura local project create hasura-sample --json`}
>

## Create a new project

This will create a new project directory with the name `hasura-sample` in the current directory and create a blank
`metadata.json` file.

</CodeStep>

<CodeStep
language="json"
code={`{
  "kind": "simpleSampleOpenDDS",
}
`}
>

## Author the metadata

Using [Open DDS](/getting-started/how-it-works#open-dds), author the metadata. This is where we'll define our data
connectors, data sources, and models.

</CodeStep>

<CodeStep
language="bash"
code={`hasura cloud project create`}
output={`Creating project... ok\nName: hasura-sample\nID: 12345678-1234-1234-1234-123456789012\nConsole URL: https://cloud.hasura.io/v3/project/12345678-1234-1234-1234-123456789012`}
>

## Create a new Hasura Cloud project

You'll have a near-instantly generated a Hasura Cloud project and be returned the name, ID, and Console's URL ðŸŽ‰

</CodeStep>

<CodeStep
language="bash"
code={`hasura cloud -p <PROJECT_ID> -m metadata.json`}
output={`Creating build... ok\nBuild ID: 12345678-1234-1234-1234-123456789012\nGraphQL API Endpoint: https://hasura-sample.ddn.hasura.app/graphql`}
>

## Create a new build

Using the project ID from the previous step, create a new build. This will upload your metadata to the Hasura Cloud and
give you a GraphQL API endpoint which you can use to query your data.

</CodeStep>
